# Windows 原生一键部署（离线考场）

本文档面向**考场管理机**与**发版构建机**。考场是**隔离局域网**，安装与运行过程**不得访问互联网**。

## 角色分工

| 环境 | 是否联网 | 操作 |
| --- | --- | --- |
| 发版机构 / 开发机 | 可以 | `fetch-runtimes.ps1` 下载 Node、PostgreSQL、VC++ → **裁剪非运行时组件** → `package.ps1` 生成 `LAN-Exam-win-v<版本>.zip` |
| 考场管理机 | 不可以 | U 盘拷贝 zip → 解压 → **以管理员运行 `setup.bat`** → 托盘 / 桌面快捷方式打开管理台 |

安装包**自包含**：Node、PostgreSQL 便携版、`vc_redist.x64.exe`、应用与迁移脚本。发版时会自动**裁剪 EDB Postgres 包中的 pgAdmin、文档、头文件等非运行时目录**，以及 Node 便携包自带的 `npm/node_modules`，以缩短考场解压时间。现场**禁止**引导考官访问微软官网或在线下载依赖。

## 发版构建（有网，考前完成）

在仓库根目录（Windows x64，已安装 Node 20+、pnpm 9+、.NET 8 SDK、Inno Setup 6）：

```powershell
cd E:\programs\LAN_exam
.\scripts\windows\package.ps1
```

发版前在仓库根目录维护 **`VERSION`**（如 `1.6.0`），`package.ps1` 会据此命名安装包并写入绿色目录。

产出：

- `dist\LAN-Exam-win-v<版本>.zip`（例如 `LAN-Exam-win-v1.6.31.zip`，**考场推荐**；v1.6.31 起解压约 **0.7 GB / ~1.6 万文件**，较旧版约减半）
- `dist\lan-exam-win\`（未压缩绿色目录，可本地调试或备份）
- `dist\LAN-Exam-Setup-v<版本>.exe`（可选：加 `-WithInstaller` 时产出，Inno 压缩较慢）

```powershell
# 默认：绿色 zip（跳过 Inno，通常几分钟内完成）
.\scripts\windows\package.ps1

# 仍需传统安装包时
.\scripts\windows\package.ps1 -WithInstaller
```

可选仅组装应用（不下载运行时）：

```powershell
.\scripts\windows\build-release.ps1
```

## 考场部署（无外网）

1. 将 `LAN-Exam-win-v<版本>.zip` 拷入 U 盘，复制到管理机并解压（建议 `D:\LAN-Exam`）。
2. **右键 `setup.bat` →「以管理员身份运行」**（首次必须）；脚本会：
   - 本地静默安装捆绑的 VC++ 运行库
   - 生成 `.env`（含随机 `SESSION_SECRET`）
   - 初始化 Postgres（`data\pg`，端口 **5434** 仅本机）
   - 执行 `prisma migrate deploy` 与种子账号 `local_exam_admin`
   - 添加入站防火墙规则 **TCP 5180**（专用 / 公用 / 域配置文件）
   - 创建桌面快捷方式并启动托盘
3. 日常启动：双击 **LAN-Exam-Tray.exe** 或桌面「局域网考试系统」。
4. 浏览器打开 `http://127.0.0.1:5180/admin`（免登录管理台）。

解压目录内另有 `考场使用说明.txt` 供现场查阅。

### 可选：Inno Setup 安装包

若使用 `package.ps1 -WithInstaller` 产出的 `LAN-Exam-Setup-v<版本>.exe`，流程与旧版相同：双击安装向导即可（内部仍调用同一套 `setup` 脚本逻辑）。

## 考前准备

### 1. 启动与健康检查

1. 双击桌面快捷方式，托盘启动服务。
2. 等待管理台自动打开，或手动访问 `http://127.0.0.1:5180/admin`。
3. 若需确认服务就绪，可在本机执行：

```powershell
curl http://127.0.0.1:5180/health
```

期望返回：`{"status":"ok"}`

### 2. 导入本场资源

在管理台完成以下导入：

- 客观题：题库管理 → 客观题
- 操作题：题库管理 → 操作题
- 操作题：题库管理 → 操作题
- 名单：名单管理

| 模块 | 需要文件 | 规则 |
| --- | --- | --- |
| 客观题 | `.xls` / `.xlsx` / `.csv` | 使用 `templates/题库导入模板.xlsx` |
| 名单 | `.xls` / `.xlsx` / `.csv` | 使用 `templates/名单导入模板.xlsx`；列为“姓名 / 单位 / 身份证号” |
| 操作题 | Word 题目 + `.xls` / `.xlsx` 答题卡 + 可选附件 | 参考 `templates/操作题导入模板.xlsx`；答题卡工作表名 `答题卡`，列“题号 / 答案 / 分值” |

重要说明：

- 切换到免登录后，旧 `teacher_admin` 数据**不会自动迁移**。
- 从 Docker Compose 或旧 session 环境升级时，请按本场需求**重新导入**题库与名单。

### 3. 检查设置

在 **设置** 页确认：

- 是否需要向学员显示座位表
- 是否需要先执行“清除全部数据”清掉上一场业务数据

“清除全部数据”会删除考试、题库、名单、操作题批次及相关文件，但会保留设置项本身。

### 4. 创建考试

在 **考试管理** 中新建考试：

1. 输入考试名称
2. 选择考试内容模块，可多选：
   - 客观题
   - 操作题
   - 操作题
3. 为每个启用模块选择对应批次
4. 选择名单批次
5. 设定开始 / 结束时间
6. 保存后进入考试详情页

开始考试前，考试状态为 `DRAFT`。点击“开始考试”后，名单内学员才能进入答题。

## 考中操作

### 学员访问

- 托盘菜单可使用 **“复制学员地址”**
- 或白板告知学员：`http://<管理机局域网IP>:5180/exam/login`

学员使用**姓名 + 身份证号**登录。

### 座位表

- 若设置页已开启座位表，学员登录页与考试详情会显示座位安排。
- 座位按考试随机分配；第一次读取座位表时系统会自动生成座位号。

### 作答行为

- 客观题 / 操作题会自动保存
- 操作题在考试端显示 Word 题目全文，并提供 Excel 答题卡下载 / 填写；每空还可上传或粘贴截图（PNG / JPEG / WebP，每空最多 5 张，单张默认不超过 5MB）作为作答佐证，**不参与自动评分**
- 操作题需上传 `.doc` / `.docx` 作答文件后才能交卷
- 作答页显示距离计划结束时间的倒计时；**到达计划结束时间后系统会自动提交试卷并结束考试**（已保存的客观/操作题作答计分，未答按 0 分）
- 已交卷学员会进入“提交成功”页；若考试尚未被系统或考官结束，可继续等待

## 考后操作

### 1. 结束考试

到达计划结束时间后，系统会自动收卷并结束考试，**一般无需考官到点再点“结束考试”**。若需提前收卷，可在考试详情页点击“结束考试”（会先对未交卷学员强制交卷，再结束）。结束后：

- 学员不能再继续保存或提交
- 已交卷学员进入只读结束页
- 未交卷学员会看到“考试已结束，未提交答案不计成绩”的提示

### 2. 导出与下载

在考试详情页：

- 点击“导出成绩与明细”下载 Excel
- 若考试包含操作题，可点击“导出操作题截图”下载 ZIP（仅含**已交卷**学员的截图；ZIP 内按学员分文件夹 `姓名_身份证后4位`，文件名为 `第一题01`、`第一题02`、`第二题01`…，题号与答题卡一致）

计分说明：

- 客观题 / 操作题按答题卡自动计分；操作题截图仅作佐证，不计入分数
- 混合考试总分仅累计自动评分部分

### 3. 下一场前清场

若下一场考试需要全新业务数据：

1. 进入 **设置**
2. 执行 **“清除全部数据”**
3. 重新导入下一场所需题库、题目批次和名单

若只是短暂停考而下一场继续使用同一批数据，可保持托盘运行，不必退出系统。

## 网络与安全

- **管理台**：仅 `http://127.0.0.1:5180/admin`
- **管理 API / 认证 API**：仅 loopback，考试机请求应返回 **403**
- **学员端**：`http://<管理机IP>:5180/exam/*`
- Postgres **5434** 不暴露局域网；仅 **5180** 对 LAN 开放学员流量
- HTTP 仅限机房内网；勿暴露公网
- 无人看管管理机等同开放管理权限

## 安装后目录

```
D:\LAN-Exam\
├── LAN-Exam-Tray.exe
├── setup.bat            # 首次配置（管理员）
├── start.bat / stop.bat / install.bat / open-admin.bat
├── 考场使用说明.txt
├── runtime\   (node, postgres, vcredist)
├── app\       (server-bundle 运行时依赖 + dist, web dist, prisma)
├── data\
│   ├── pg\    (数据库，升级保留)
│   └── ...    (上传文件、答卷、衍生文件)
├── logs\
└── .env
```

关闭托盘窗口只会**最小化到托盘**，服务继续运行；仅托盘“退出系统”才调用 `stop.bat` 停止 Node 与 Postgres（`scripts\stop-postgres.ps1` 会先 `pg_ctl stop`，失败则结束本机 5434 监听进程）。若退出后仍见 `postgres.exe`，查看 `logs\stop.log`。

## 日志与排障入口

- 应用日志：`logs\app.log`
- 健康检查：`http://127.0.0.1:5180/health`
- 配置文件：`.env`
- 防火墙规则：`wf.msc` 中检查 “LAN Exam TCP 5180”

## 常见问题

| 现象 | 排查 |
| --- | --- |
| 管理台打不开 | 托盘是否运行；`logs\app.log`；本机 `curl http://127.0.0.1:5180/health` |
| 考试机无法打开学员页 | 防火墙专用网络是否启用；管理机 IP 是否变化；`wf.msc` 检查 5180 规则 |
| 考试机可打开 `/admin` | 页面提示“请在本机打开”属预期；`/api/admin/*` 应 403 |
| 操作题导入失败 | 检查答题卡是否为 `.xls/.xlsx`，工作表名是否为 `答题卡`，列是否为“题号 / 答案 / 分值” |
| 上传作答文件失败 | 检查 `.doc` / `.docx` 文件大小与上传限制；必要时查看 `.env` 中 `MAX_PRACTICAL_*` |
| 操作题截图上传失败 | 确认格式为 PNG / JPEG / WebP、每空不超过 5 张、单张大小；必要时查看 `.env` 中 `MAX_FILLIN_SCREENSHOT_BYTES` 与 `MULTIPART_MAX_FILE_BYTES` |
| 导入后看不到旧数据 | 当前只显示 `local_exam_admin` 下的数据；旧 `teacher_admin` 数据需重新导入 |
| 端口占用 | 修改 `.env` 中 `WEB_PORT` 并同步调整防火墙规则 |

## 升级

替换 `app\` 与 `runtime\`（保留 `data\`、`logs\`、`.env`），再运行 `install.bat`。升级流程仅补充 migrate，不重新 `initdb`。

## 开发机验证免登录

根目录 `.env`：

```env
ADMIN_AUTH_MODE=disabled
VITE_ADMIN_AUTH_MODE=disabled
```

```bash
pnpm db:up && pnpm db:migrate && pnpm db:seed
pnpm dev
```

访问 `http://127.0.0.1:5180/admin` 应直达仪表盘。生产单进程模式下，`NODE_ENV=production` 启动后由服务端在 `5180` 托管静态资源。
