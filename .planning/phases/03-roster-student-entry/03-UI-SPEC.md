# Phase 3 — UI Design Contract

> Visual and interaction contract for **名单与学生入场**（名单 Excel 导入与检索、学生姓名+身份证强校验登录、考试准备等待页）。Generated by gsd-ui-researcher; verify with gsd-ui-checker before planning execution.

**Scope:** 管理端 `/admin/roster`；仪表盘「名单」入口启用；学生端 `/exam/login`、`/exam/waiting`（根路径 `/` 重定向至登录）；`GET /api/student/me` 驱动准备页展示。中文界面。延续 Phase 1/2 `AdminLayout`、shadcn new-york zinc、文案语气。

**Out of scope (UI):** 组卷、答题、提交、成绩导出（Phase 4）；学生端考试界面；管理端「考试」卡片仍 disabled。

## URL & Layout Shell

| Route | Surface | Auth |
|-------|---------|------|
| `/` | 重定向至 `/exam/login`（`replace`） | 公开 |
| `/exam/login` | 学生身份验证（姓名 + 18 位身份证） | 公开 |
| `/exam/waiting` | 考试准备等待页（D-06） | 已建立学生 session |
| `/admin` / `/admin/dashboard` | 仪表盘；「名单」卡片 → `/admin/roster` | 教师已登录且已完成改密 |
| `/admin/roster` | 名单导入 + 检索列表 | 同上 |

**Admin shell:** `/admin/roster` 使用现有 `AdminLayout`（顶栏 56px / `h-14`、主内容 `max-w-[960px]`、`px-4 py-8`）。**无** 二级侧边栏。

**Student shell:** 登录与准备页 **不使用** `AdminLayout`；全屏居中卡片布局（对齐 Phase 1 `AdminLogin`），背景 `bg-background`，无顶栏产品导航至 `/admin`（教师 URL 由部署文档提供）。

**页面信息架构 — 管理端 `/admin/roster`（自上而下）：**

1. 页头：标题 + 说明 + 返回仪表盘链接
2. 导入区（Card）
3. 导入结果区（成功摘要 / 错误表 — 仅在一次导入尝试后有内容）
4. 名单列表区（搜索 + 表格 + 分页）

**页面信息架构 — 学生端：**

| 页面 | 结构 |
|------|------|
| `/exam/login` | 单卡片：产品副标题 + 表单（姓名、身份证号）+ 主 CTA |
| `/exam/waiting` | 单卡片：身份确认区（姓名、身份证号只读展示）+ 等待文案 + 退出 |

## Phase Surfaces

### S0 — 仪表盘「名单」入口 (`/admin`)

| Element | Spec |
|---------|------|
| 名单卡片 | `Link` → `/admin/roster`；`hover:border-primary/50`，`focus-visible:ring-2 ring-ring`；副标签「进入」（与题库卡片一致） |
| 题库卡片 | 保持 Phase 2 可点击行为 |
| 考试卡片 | 保持 `opacity-60`、`cursor-not-allowed`、`即将开放` |
| 仪表盘说明文案 | **默认保留** Phase 2「管理功能筹备中」段落；可选将「名单管理」从「将在后续版本开放」改为已开放表述 — **本契约默认不改主文案**，仅名单卡片可点 |

### S1 — 名单管理页 (`/admin/roster`)

**Purpose:** ROST-01；教师下载模板、上传名单、查看导入结果、按姓名或证件号检索。

| Element | Spec |
|---------|------|
| 页面标题 | Heading（20px）：「名单管理」 |
| 说明 | Body（16px）：「请使用官方 Excel 模板批量导入考生名单。导入前可下载模板，姓名与身份证号须与证件一致（除首尾空格外须完全一致）。」 |
| 返回 | Body 链接「← 返回仪表盘」→ `/admin`（14px semibold，`text-primary`） |

### S2 — 名单导入区（页内 Card #1）

| Element | Spec |
|---------|------|
| Card 标题 | Label（14px semibold）：「导入名单」 |
| 下载模板 | `Button` `variant="outline"`，图标 `Download`；`GET /api/admin/roster/template` → `名单导入模板.xlsx` |
| 上传区 | 与 Phase 2 题库一致：虚线框最小高度 **160px**，`border-2 border-dashed`；图标 `Upload` + 主文案 + 辅助行「仅支持 .xlsx，单文件不超过 5MB」 |
| 交互 | 点击 / 拖拽；选中后显示文件名 +「更换文件」 |
| 文件校验（前端） | 扩展名 `.xlsx`；`size ≤ 5_242_880`；不合规 Alert destructive，不请求 |
| 主 CTA | Primary：「开始导入」；无文件时 `disabled` |
| 提交中 | 按钮 loading + 上传区 `pointer-events-none opacity-60`；文案「正在导入…」 |

**布局间距：** Card `p-6`；下载与上传区间距 16px；上传区与 CTA 间距 24px。

**实现模式：** 复用 Phase 2 `ImportDropzone` 交互与视觉；可抽 `RosterImportDropzone` 或参数化共用组件，**视觉规格与题库导入区一致**。

### S3 — 名单导入成功反馈

**Trigger:** `POST /api/admin/roster/import` 返回 `200` 且 `ok: true`。

| Element | Spec |
|---------|------|
| Toast | `sonner` success：见 Copywriting |
| 摘要 Card | 可选左边框 `border-l-4 border-l-primary`；Heading 行「导入成功」+ Body 统计：`成功导入 {importedCount} 人`、`跳过示例行 {skippedCount} 条`（为 0 隐藏）、`来源文件：{fileName}` |
| 列表刷新 | 成功后 `refetch` 名单列表（重置第 1 页）；搜索框保留当前关键词（若有） |

**禁止：** 失败态展示「部分成功」人数（与 ALL_OR_NOTHING / 整批拒绝策略一致）。

### S4 — 名单导入失败反馈

**Trigger:** `400` 且 `errors[]`（整批未写入）。

| Element | Spec |
|---------|------|
| Toast | `sonner` error：见 Copywriting |
| Alert | destructive：「未写入任何名单记录」 |
| 错误表 | `Table` 列：**行号**、**列名**、**原因**；超过 50 行时表体 `max-h-[320px]` 可滚动 |
| 表上方 Body | 「共 {n} 处错误，请修正 Excel 后重新导入。」 |

### S5 — 名单列表与检索（页内 Card #2）

| Element | Spec |
|---------|------|
| Card 标题 | 「名单列表」 |
| 搜索 | 单行 `Input`，`placeholder="按姓名或身份证号搜索"`；右侧或下方 Primary/outline「搜索」；**Enter** 提交；清空按钮（有内容时显示 `X` 或「清空」link） |
| 表格列 | **姓名**｜**身份证号**（见脱敏规则）｜**导入时间**（`yyyy-MM-dd HH:mm`） |
| 脱敏（管理端默认） | 列表列显示：`前6位 + ******** + 后4位`（不足 18 位则 Body 显示「—」）；**搜索命中仍可按完整证件号检索**（服务端支持）；详情/导出全号本阶段 **无** 单独 Dialog |
| 分页 | 每页 **20** 条；底部分页文案与 Phase 2 一致 |
| 空状态（无记录） | 见 Copywriting |
| 空状态（搜索无结果） | Heading「未找到匹配记录」+ Body「请检查姓名或身份证号是否输入正确。」 |
| 加载态 | Skeleton 3 行或 Spinner +「加载名单…」 |

### S6 — 学生登录 (`/exam/login`)

**Purpose:** AUTH-02 入口；D-01、D-03、D-04。

| Element | Spec |
|---------|------|
| 视觉锚点 | Display（28px）「考生登录」为页面主焦点 |
| 副标题 | Body：「局域网考试系统」 |
| 字段 | `姓名`（`text`，`autoComplete="name"`）；`身份证号`（`text`，`inputMode="numeric"`，`maxLength={18}`，`autoComplete="off"`，**非** password 类型以便机房核对输入） |
| 填写说明 | Label 下方 muted 一行：「请输入与考试名单一致的姓名和 18 位身份证号。」 |
| 主按钮 | 全宽 Primary，文案见 Copywriting |
| 提交中 | loading + 字段 disabled |
| 前端校验 | 姓名 trim 后非空；身份证 trim 后 **18 位格式 + 校验码**（与后端 D-04 一致）；格式非法时 **字段级或表单级错误**，文案见 Copywriting，**不发起** verify 请求 |
| 后端失败 | 表单顶部 destructive `Alert`：**单一笼统文案**（不区分姓名错/证件错/不在名单 — ROADMAP SC #3、CONTEXT 灰区 3） |
| 成功 | 建立学生 session 后 `navigate('/exam/waiting', { replace: true })` |
| 键盘 | Enter 在身份证框聚焦时提交 |

**Card:** `max-w-[400px]`，全宽小屏，`px-4 py-16` 外层留白，Card 内边距与 Phase 1 登录一致（`CardHeader` + `CardContent`）。

**禁止：** 页内链接至 `/admin`；页脚不展示教师入口。

### S7 — 考试准备等待页 (`/exam/waiting`)

**Purpose:** D-06、D-07、D-08；展示 `GET /api/student/me` 返回的完整姓名与身份证号。

| Element | Spec |
|---------|------|
| 页面标题 | Display：「考试准备」 |
| 身份区 | `Card` 或带边框区块：Label 行「姓名」+ Body **完整姓名**；Label 行「身份证号」+ Body **完整 18 位**（`font-mono` 或 `tracking-wide` 提升可读性，**禁止**脱敏） |
| 等待文案 | Body 16px，居中或身份区下方，**精确文案**：「请等待监考教师开始考试」 |
| 辅助说明 | Muted Body（可选）：「监考教师开始考试后，本页将自动进入答题界面。」— **Phase 4 前可省略或保留为预期说明**；若保留须 muted 且不含可操作按钮 |
| 退出 | 全宽或右下 `Button` `variant="outline"`：「退出」；点击 `POST /api/student/logout`（或等价）→ 清 session → `/exam/login`；**无需**二次确认（对齐教师退出） |
| 刷新 | 刷新页面保持登录直至 session 过期或用户退出（D-08） |
| 加载 | 首次进入 `GET /api/student/me` 期间全卡片区 Spinner +「加载考生信息…」 |
| 错误 | 401 → 重定向 `/exam/login`；5xx → Alert +「重试」 |

**Card:** 宽 `max-w-[440px]`（略宽于登录以容纳 18 位证件号单行展示）；小屏允许身份证号 `break-all`。

### S8 — 学生路由守卫

| State | Behavior | UI |
|-------|----------|-----|
| `checking` | 校验学生 session | 全屏居中 `Spinner` +「正在验证考生身份…」 |
| `unauthenticated` | 访问 `/exam/waiting` 无 session | 重定向 `/exam/login` |
| `authenticated` | 访问 `/exam/login` 已有 session | 重定向 `/exam/waiting` |
| `sessionExpired` | 学生 API 401 | Toast「登录已过期，请重新验证身份。」→ `/exam/login` |

**与教师守卫隔离：** 学生路由 **不** 包裹 `AdminRoute`；教师 cookie 与学生 cookie 互不影响。

## Color

继承 `index.css` / Phase 1–2 契约。

| Role | Value | Phase 3 usage |
|------|-------|---------------|
| Dominant (60%) | `#F8FAFC` (`--background`) | 学生/管理页背景 |
| Secondary (30%) | `#FFFFFF` (`--card`) + `#E2E8F0` (`--border`) | 卡片、表格、上传虚线框 |
| Accent (10%) | `#2563EB` (`--primary`) | 见下方保留列表 |
| Destructive | `#DC2626` (`--destructive`) | 导入失败、验证失败 Alert |
| Muted text | `#64748B` (`--muted-foreground`) | 辅助说明、空状态 |

4. 输入框 `focus-visible` ring（沿用全局）

**禁止：** accent 用于表格行背景；等待页 **不得** 用 destructive 色强调身份证号（非错误态）。

**已安装（Phase 1–2）：** `alert`, `button`, `card`, `form`, `input`, `label`, `spinner`, `table`, `dialog`, `badge`, `select`

|------|------|
